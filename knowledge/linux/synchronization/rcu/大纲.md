# 第1章\_RCU\_专题大纲

## 1.1\_专题定位

RCU（Read-Copy-Update）是 Linux 内核中面向读多写少场景的通用同步机制。它将“读者如何安全观察对象”、“更新者如何发布新状态”和“旧对象何时可以回收”拆成不同的正确性问题。

本专题的权威机制正文放在当前目录；对象生命周期、数据结构和特定版本源码的内容仍保留在各自的本质归属中，本文只负责组织阅读顺序。

## 1.3\_学习主线

阅读时始终围绕以下四个问题：

1. RCU 依赖哪些硬件与编译器机制，哪些正确性必须由内核软件完成？
2. Tree、PREEMPT、Tiny、SRCU 和 Tasks RCU 分别等待哪些读者？
3. Tree RCU 如何从 CPU/QS/EQS 与被抢占任务中得出 GP 已完成？
4. 回调如何从注册、关联 GP、分段前移到最终执行？
5. 最后才问 API 如何表达上述运行机制。

## 1.4\_完成标准

完成本专题后，应当能够：

- 画出读者、更新者、宽限期和回收回调的时序关系。
- 区分 `rcu_assign_pointer()`、`rcu_dereference()`、`synchronize_rcu()`、`call_rcu()` 和 `kfree_rcu()` 的职责。
- 解释 `list_del_rcu()` 为什么不等于对象立即不可见，更不等于可以立即释放。
- 在普通 RCU、SRCU、seqcount/seqlock 和传统锁之间做出有依据的选择。
- 检查 RCU 与 kref、哈希表或链表组合时的指针生命周期错误。
