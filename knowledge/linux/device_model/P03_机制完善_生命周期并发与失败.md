# 第3章\_设备模型机制完善\_生命周期并发与失败

朴素注册—匹配模型还要面对：对象遍历期间被删除、probe 并发、依赖资源未就绪、父子设备删除顺序、用户空间仍持有 sysfs 引用、驱动模块正在卸载以及 probe 中途失败。

| 反例 | 被迫增加的机制 |
| --- | --- |
| 对象从集合删除但仍被引用 | kobject/kref 与 release 回调 |
| supplier 尚未就绪 | deferred probe |
| probe 部分成功后失败 | 反向回滚与 devres |
| consumer/supplier 删除或 PM 顺序 | device link |
| 内核对象需用户空间可见 | kobject/kernfs/sysfs 与 uevent |
