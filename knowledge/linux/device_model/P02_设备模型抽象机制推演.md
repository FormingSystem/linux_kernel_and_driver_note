# 第2章\_设备模型抽象机制推演

从第一章约束可以推出四类对象：设备、驱动、总线和层次对象。总线保存两个集合并提供匹配策略；设备和驱动无论谁先注册，都要触发一次对另一集合的搜索。

```text
设备注册 → 进入总线设备集合 → 遍历驱动 → match → probe → bound
驱动注册 → 进入总线驱动集合 → 遍历设备 → match → probe → bound
```

还必须分开四个动作：对象初始化、对系统可见、建立绑定、最终释放。注册失败只能撤销已经完成的阶段；从集合删除也不等于内存立即可释放。
