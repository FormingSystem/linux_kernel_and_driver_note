# 第1章\_Linux\_Input\_子系统专题大纲

## 1.1\_专题定位

Input 子系统把键盘、鼠标、触摸屏、摇杆等不同硬件产生的数据统一为 `type/code/value` 事件流。本专题不按 API 名称堆目录，而沿着“直接暴露硬件的缺口 → 事件模型 → Linux 状态与通信实现 → 驱动生命周期 → 复杂设备与异常路径 → 验证”的因果链展开。
